# تقرير حالة الباك إند - مشروع MedRec

## 🟢 نظرة عامة على النظام (ما تم إنجازه)

يمثل الباك إند (Backend) المحرك الأساسي الذكي لمشروع MedRec؛ حيث يتولى إدارة الحسابات، البيانات الطبية المعقدة، وتحليل صور الأدوية. تم تأسيس بنية تحتية قوية قابلة للتوسع تعتمد على:

### 1. **نظام المصادقة والتأمين (Authentication)**
تم تأمين النظام باستخدام `JWT - JSON Web Tokens` مما يضمن عدم حفظ بيانات تسجيل الدخول في الجلسات بشكل تقليدي، بل عبر توكنز مشفرة تتجدد تلقائياً، وهو المعيار الأفضل لتطبيقات الموبايل (Flutter/React Native).

### 2. **قاعدة بيانات الأدوية (Medicine Model)**
جداول متقدمة لتخزين المعلومات بدقة متناهية:
- **المواد الفعالة (Active Ingredients)** وتركيزاتها وهو أمر حاسم في بدائل الأدوية.
- **التفاصيل الطبية:** الجرعات، الآثار الجانبية، والتحذيرات.
- **البيانات التجارية:** مثل الأسعار، مناطق التوافر، التعبئة الدوائية، والشركة المصنعة.

### 3. **معالجة الذكاء الاصطناعي (AI Intermediary)**
الباك إند حالياً مجهز بملف جسر (`core/ai_service.py`) وظيفته استلام الصورة من المستخدم عبر الـ API، وتمريرها لموديل الـ Machine Learning الداخلي، ثم استلام النتيجة (الدواء المكتشف + نسبة الثقة Confidence) وربطها بقاعدة البيانات لدينا بسلاسة.

## 🔌 تفاصيل الـ APIs المتاحة حالياً (الدليل الشامل)

هنا تجد تفصيلاً معمقاً لجميع نقاط الاتصال (Endpoints)، مع توضيح المدخلات والمخرجات:

### 💊 **أولاً: مسارات الأدوية والبحث الدقيق (Medicines & Advanced Search)**

#### `GET /api/medicines/`
* **الوصف:** استعراض قائمة الأدوية (يدعم الـ Pagination).
* **إمكانيات الفلترة:** يمكنك إرسال باراميترز للبحث المحدد، مثل: `?category=مسكنات` أو ترتيب النتائج بالاسم أو السعر.

#### `GET /api/medicines/{code}/`
* **الوصف:** استرجاع **الملف الشامل** لدواء محدد بناء على الـ Code الشريطي الفريد (مثل `MED001`).
* **التفاصيل المرجعة:** يعيد JSON ضخم يحتوي على (المواد الفعالة، التركيزات، مناطق توافر الدواء، البدائل النصية، الأعراض الجانبية، التحذيرات الطبية، والتسعير).

#### `GET /api/medicines/search/?q={query}`
* **الوصف:** **محرك بحث متقدم وذكي جداً.**
* **طريقة العمل:** بمجرد كتابة حرف أو كلمة في `{query}`، يقوم الباك إند بالبحث المدمج بالتوازي في:
  - الاسم العربي (`name_ar`).
  - الاسم الإنجليزي (`name_en`).
  - **المواد الفعالة (`active_ingredients`)**.
  - **قائمة البدائل (`alternatives`)**.
* **الفائدة:** إذا بحث المريض عن مادة "Paracetamol"، ستظهر له مباشرةً جميع الأدوية التي تحتوي على هذه المادة (مثل بانادول أو فيفادول) حتى وإن لم يبحث عن اسم الدواء التجاري.

#### `GET /api/medicines/{code}/alternatives/` *(جديد)*
* **الوصف:** نقطة اتصال ذكية لإيجاد **الأدوية المطابقة** لدواء معين.
* **طريقة العمل:** يقوم باستخراج المادة الفعالة من الدواء المطلوب، ثم يبحث في قاعدة البيانات ويستخرج لك قائمة بكل الأدوية الجنيسة (Generics) التي تملك نفس المادة الفعالة بالضبط للإقتراح على المستخدم في حالة عدم توافر دوائه الأصلي.

### 📷 **ثانياً: مسارات الكاميرا / رفع وتحليل الصور (Image Upload Endpoints)**

#### `POST /api/uploads/new/`
* **الوصف:** الدالة الرئيسية لاستقبال ومعالجة الصور المرفوعة من الكاميرا وتتطلب تسجيل دخول `(Bearer Token)`.
* **المدخلات (Payload):** ملف عبر `multipart/form-data` كـ `image`.
* **العملية الداخلية للمسار (Internal Process):**
  1. يتسلم الصورة ويحفظها بأمان داخل خادم المشروع في مجلد يحمل تاريح اليوم.
  2. يرسلها لخدمة (`ai_service.py`) لاستخراج التوقعات.
  3. يقوم بإنشاء (Match) لنتيجة الذكاء الاصطناعي مع الداتا بيس لمعرفة الدواء.
* **المخرجات (Response):** يعيد JSON يحتوي على:
  - `image_url`: الرابط الدائم للصورة للمعاينة.
  - `medicine_details`: بيانات الدواء المكتشف كاملاً.
  - `confidence`: نسبة الدقة (مثال: `0.92` أي بمعدل دقة 92%).

#### `GET /api/uploads/`
* **الوصف:** عرض **السجل الطبي السالف (History)** العائد للمستخدم الحالي فقط.
* **الفائدة:** يطبع بيانات الصور السابقة التي رفعها المستخدم مع نتيجة التحليل وتاريخ الفحص، مفيدة لعمل قسم "سجل الفحوصات" داخل تطبيق الموبايل.

#### `GET /api/uploads/{id}/` *(جديد)*
* **الوصف:** جلب فحص صورة معين بتفاصيله بواسطة رقم الفحص.

#### `DELETE /api/uploads/{id}/` *(جديد)*
* **الوصف:** السماح للمستخدم بمسح فحص/صورة سابقة من سجله تماماً لحفظ الخصوصية.

---

### 🔐 **ثالثاً: مسارات المصادقة (Auth Endpoints)**
* `POST /auth/register/` ➔ إنشاء الحساب.
* `POST /api/token/` ➔ تسجيل الدخول والحصول على `Token`.
* `POST /api/token/refresh/` ➔ تجديد الجلسة دون إزعاج المستخدم.
